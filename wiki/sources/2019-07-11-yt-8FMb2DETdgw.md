## Summary

A hands-on Webflow tutorial from the Blind/Futur "Building A Brand" series, filed under the
Hamilton Family Brewery rebrand. The narrator (a Futur/Blind designer, self-described "not
an expert") walks through the four Webflow techniques they picked up while building the
Hamilton site in-house — because the client's web-development budget was small, the team
wanted designers to build the site themselves without shipping to a developer. The four
tips: (1) truly responsive columns with Flexbox, (2) themed custom content types via the
Webflow CMS, (3) using symbols as a reusable library of page-layout parts, and (4) building
quick hover animations with the interactions panel. The team liked Webflow enough that they
say they'll migrate The Futur's own site from WordPress to Webflow. Ends with a Webflow
sign-up plug that supports The Futur.

## Key claims

_All attributed to an unnamed Futur/Blind team member — NOT Chris Do. Context only._

- [2019-07-11] The team chose Webflow for the Hamilton Family Brewery rebrand because the
  client's web-development budget "isn't much," so they needed designers to build the site
  in-house without handing off to a developer. (paraphrase)
- [2019-07-11] Webflow's learning curve, initially "a little bit overwhelming," became much
  less steep with use; the narrator praises how it translates CSS into a visual tool
  designers are used to. (paraphrase)
- [2019-07-11] Tip 1 — Flexbox: set the parent container to flex and size child columns by
  percentage (not fixed pixels) so columns scale and stack responsively; centering is one
  click, "as easy as Photoshop." Webflow later added CSS grid, which the narrator now
  recommends going forward and notes flex and grid can be used together. (paraphrase)
- [2019-07-11] Tip 2 — Webflow CMS collections let non-developers manage repeating content;
  they built a "beers" collection with fields (beer name, alcohol content, short
  description, label color) and one styled template so founder Josh can add/edit/remove
  beers without touching page design. Framed as far easier than a WordPress custom post
  type. (paraphrase)
- [2019-07-11] Tip 3 — Symbols: editing a symbol on one page propagates the change to all
  pages. The narrator admits an unintended use — building a personal library of reusable
  layout parts (e.g. a two-column grid) in the symbol menu, then dragging one in and
  "disassembling" it so each instance gets its own content — which saved a lot of time.
  (paraphrase)
- [2019-07-11] Tip 4 — Animations: hover interactions (e.g. a subtle ~20° z-axis rotation on
  hover, reset to zero on hover-out) are quick to build in the interactions panel with
  easing, avoiding the usual engineer-then-hand-to-developer overhead. (paraphrase)
- [2019-07-11] The full Hamilton Family Brewery site was built in "just a few days" with no
  developers and no hand-written code; Webflow can export clean front-end code in a few
  clicks if a developer ever needs it. (paraphrase)
- [2019-07-11] The team liked Webflow enough that they plan to migrate The Futur's own
  website from WordPress to Webflow. (paraphrase) — Futur org context.
- [2019-07-11] Design philosophy aside: "I'm all about being functional over being flashy,"
  but tasteful animation adds visual delight. (quote/paraphrase — team member, not Chris.)
