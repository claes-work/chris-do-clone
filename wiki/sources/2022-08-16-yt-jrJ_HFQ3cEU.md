An ~8-minute educational overview of the **heuristic evaluation** — a UX research method for
assessing a website or app against predetermined, universal usability criteria (rooted in
behavioral psychology) without needing to recruit users. Brethwaite frames it as a fast,
low-friction way for a designer to prove the value of UX to a client and win business. He
covers what it is, why a designer should care, how it is applied, and lists three named
heuristic frameworks. He repeatedly plugs a forthcoming course ("in the future"). This is
craft/tutorial content about a design methodology; it is retained here only as a
`wiki/sources/` record of a channel upload, not as subject knowledge.

## Key claims

All claims below are attributed to **Vincent Brethwaite** (guest), dated 2022-08-16,
paraphrased. NOT subject-attributed — for context only.

- A heuristic evaluation is an assessment based on mental shortcuts / criteria rooted in
  behavioral psychology, commonly used in UX design. (Brethwaite, 2022-08-16)
- It is one of the few UX methods that does **not** require engagement with actual users or
  customers, so no participant recruiting is needed. (Brethwaite, 2022-08-16)
- It can be run at any point in the design journey — before a kickoff meeting to establish
  scope, or as a checkpoint to evaluate existing designs. (Brethwaite, 2022-08-16)
- It reduces friction at project start by quickly surfacing what is and isn't working on a
  site, producing clear data a designer can present to a client to justify being hired.
  (Brethwaite, 2022-08-16)
- It doubles as an educational device: it lets the designer introduce/"sell" the power of UX
  to a client unaware of it — framed as "an appetizer before the entrée." (Brethwaite, 2022-08-16)
- A proper evaluation requires the designer to stay objective (focus on usability, not
  personal feelings), consider target users' behaviors/mindset/goals, evaluate more than one
  page, and measure all selected interface parts with the same scale/rating system.
  (Brethwaite, 2022-08-16)
- Three named heuristic frameworks: the **UX Honeycomb** (Peter Morville, 2004 — useful,
  desirable, valuable, credible, accessible, findable, usable); the **Abby method**
  (Abby Covert, 2012 — findable, accessible, clear, communicative, useful, credible,
  controllable, valuable, learnable, delightful); and the **10 Usability Heuristics**
  (Nielsen Norman Group, 1994, since updated). (Brethwaite, 2022-08-16)
- On which method to use / whether there is a standard: "it depends" — there is no single
  right way; methods can be remixed to suit the designer and task. (Brethwaite, 2022-08-16)
